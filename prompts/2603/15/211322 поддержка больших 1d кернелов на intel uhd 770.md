# Поддержка больших 1D кернелов на Intel UHD 770

## Сообщения пользователя

- начать новый промпт - поддержка запуска больших 1D кернелов на Intel UHD 770
- откати коммит Reduce Vulkan test sizes to satisfy Intel UHD 770 validation limits
- проверь что юнит тесты теперь падают
- это происходит из-за того что на некоторых GPU максимальное число рабочих групп - 65535
- теперь мы от всех видеокарт требуем хотя бы такой 65535 лимит (иначе пропускаем видеокарту и пишем об этом в лог по аналогии с другими пропусканиями видеокарт)
- но при этом хочется поддержать запуск больших 1D кернелов на таких видеокартах
- мне приходит в голову такой план (предложи лучше если придумаешь):
  - если при использовании Vulkan API код создает 1D gpu::WorkSize(256, n) - кинуть rassert и подробное сообщение что нужно сделать чтобы это исправить - создать gpu::WorkSize1DTo2D(256, n) (который создает 2D пространство) и в кернеле в месте где вызвалось бы const uint i = gl_GlobalInvocationID.x; - вызвать uint i = 2DTo1D(gl_GlobalInvocationID);
  - и естественно добавить эти две вспомогательные сущности

## Короткий лог действий

- найден коммит `c6915be`, который уменьшал размеры `vulkan.aplusb` и `vulkan.atomicAdd`
- подтверждено, что причина с `aplusb` была в oversized 1D Vulkan dispatch и лимитах `maxComputeWorkGroupCount/baseGroupX`
- отдельно выявлено, что `atomicAdd` после отката снова упирается в `maxStorageBufferRange`, это независимая проблема
- добавлен host helper `gpu::WorkSize1DTo2D(...)`
- добавлен shader helper `workSize2DTo1D(gl_GlobalInvocationID)`
- Vulkan compute path теперь явно бросает `rassert` для любого raw 1D `gpu::WorkSize(...)` с инструкцией как перейти на 2D layout
- `vulkan.aplusb` переведен на новый helper и возвращен к `n = 100 * 1000 * 1000`
- остальные Vulkan 1D compute tests тоже переведены на `gpu::WorkSize1DTo2D(...)`
- `vulkan.atomicAdd` возвращен к старому большому setup и теперь пропускает только те устройства/setup, где `maxStorageBufferRange` слишком мал
- добавлены unit tests:
  - `gpu_device.workSize1DTo2DKeepsXWithinLimit`
  - `vulkan.raw1DWorkSizeRequiresExplicit2DConversion`
- обновлен `libs/gpu/libgpu/vulkan/README.md`

## Короткий лог результатов

- Intel UHD 770 (`GPGPU_VISIBLE_DEVICES=2`, validation layers on):
  - `vulkan.aplusb` passed
  - `vulkan.raw1DWorkSizeRequiresExplicit2DConversion` passed
  - `vulkan.atomicAdd` passed
  - `vulkan.binarySearch` passed
  - `vulkan.radixSort` passed
  - `vulkan.asyncTinyWriteValueChain` passed
- all visible real devices, validation layers on:
  - `gpu_device.workSize1DTo2DKeepsXWithinLimit` passed
  - `vulkan.aplusb` passed on Intel UHD 770 and NVIDIA GTX 1650, skipped on `llvmpipe` because `maxStorageBufferRange=134217728 < 400000000`
  - `vulkan.raw1DWorkSizeRequiresExplicit2DConversion` passed
  - `vulkan.atomicAdd` passed; the `100000000` table setup is skipped only on `llvmpipe` for the same `maxStorageBufferRange` reason
