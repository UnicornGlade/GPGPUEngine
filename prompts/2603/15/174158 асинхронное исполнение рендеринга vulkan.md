# Задача

Пользователь попросил:
- начать новую задачу по асинхронному исполнению рендеринга;
- опереться на уже сделанный async compute;
- сначала изучить план и lessons learned по async compute;
- затем спланировать async render;
- задокументировать план;
- реализовать;
- добавить unit-тесты, включая performance-oriented test, который должен заметно ускориться.

## Связанные документы

- Подробный план: [docs/vulkan_async_render_plan.md](/home/polarnick/codex/GPGPUEngine/docs/vulkan_async_render_plan.md)

## Короткий журнал

- Изучен текущий async compute plan и lessons learned из profiling.
- Подтверждено, что render path сейчас синхронный и использует отдельный blocking wait в `launchRender`.
- Сформирован план: обобщить inflight launch model на compute+render, добавить async render submit, render rassert slots, render stats и новые tests.
- Реализован async render submit с bounded inflight render queue, render stats и двухслотовым raster `rassert`.
- Добавлены unit-tests:
  - `vulkan.asyncRenderBlendingChainAccumulatesCorrectly`
  - `vulkan.asyncRenderBlendingChainRespectsInflightLimit`
  - `vulkan.asyncRenderBlendingChainShowsSpeedup`
- Исправлен существующий `vulkan.aplusb` test под async compute contract: GPU error теперь проверяется на explicit sync point.
- Сняты `nsys` before/after профили на GTX 1650.
- Выполнен полный прогон `libgpu_test` на GTX 1650.
